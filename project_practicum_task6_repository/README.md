
# База данных для приложения управления настольно-ролевой игрой

Репозиторий содержит структуру данных для системы управления персонажами, игровыми кампаниями и боевыми событиями настольно-ролевой игры.

## Назначение

Структура данных используется для хранения и обработки информации о:

- персонажах игроков
- игровых кампаниях
- игровых сессиях
- боевых столкновениях
- участниках боя
- результатах бросков кубов
- правилах игры
- заметках ведущего

Данные представлены в формате CSV. Такой формат позволяет импортировать таблицы в систему NocoDB и использовать их как таблицы базы данных.

## Структура репозитория

project/
- README.md

  - characters.csv
  - campaigns.csv
  - sessions.csv
  - combat_encounters.csv
  - combat_participants.csv

  - notes.csv

## Описание таблиц

### characters

Таблица содержит информацию о персонажах.

Поля:

- id - идентификатор персонажа
- name - имя персонажа
- player - имя игрока
- race - раса
- class - класс
- level - уровень
- hp - здоровье
- mana - мана
- strength - характеристика силы
- dexterity - характеристика ловкости
- intelligence - характеристика интеллекта
- notes - дополнительные сведения

### campaigns

Таблица игровых кампаний.

Поля:

- id - идентификатор кампании
- title - название кампании
- game_master - ведущий игры
- setting - игровой мир
- start_date - дата начала
- description - описание

### sessions

Таблица игровых сессий.

Поля:

- id - идентификатор
- campaign_id - ссылка на кампанию
- session_number - номер сессии
- date - дата проведения
- summary - краткое описание

### combat_encounters

Таблица боевых столкновений.

Поля:

- id - идентификатор
- session_id - ссылка на сессию
- name - название столкновения
- location - место действия
- status - состояние боя

### combat_participants

Участники боевого столкновения.

Поля:

- id - идентификатор
- encounter_id - ссылка на бой
- character_id - персонаж
- initiative - значение инициативы
- current_hp - текущее здоровье
- status_effects - активные эффекты

### dice_rolls

Журнал бросков кубов.

Поля:

- id - идентификатор
- character_id - персонаж
- dice_formula - формула броска
- result - результат
- context - игровое действие
- timestamp - время выполнения

### rules_reference

Справочник правил.

Поля:

- id - идентификатор
- rule_name - название правила
- category - категория
- description - описание
- source - источник правила

### notes

Таблица заметок.

Поля:

- id - идентификатор
- related_type - тип объекта
- related_id - идентификатор объекта
- text - текст заметки
- created_at - дата создания

## Использование в NocoDB

1. Создать новый проект в NocoDB.
2. Добавить таблицы через импорт CSV.
3. Импортировать файлы из каталога data.
4. После импорта настроить связи между таблицами.
